# Network Create

This document describes the `network-create` endpoint, which is used to create a new network and associate a user authority with it. A user authority can be associated with at most one network.

---

## `POST /auth/network-create`

Create a new network. A user authority can be associated with at most one network.

## `POST /auth/network-create`

Create a new network. A user authority can be associated with at most one network.

### Request

`POST /auth/network-create`

**Headers:**

```json
{
  "Host": "api.bringyour.com",
  "Content-Type": "application/json",
  "Accept": "*/*",
  "Content-Length": "[calculated-length]"
}
```

**Body:**

The request body should be a JSON object containing details for the new user and network.

```json
{
  "user_name": "string",
  "user_auth": "string",
  "auth_jwt": "string",
  "auth_jwt_type": "string",
  "password": "string",
  "network_name": "string",
  "terms": "boolean"
}
```

| Field           | Type      | Description                                                                                                                                                                                                                                                                 | Possible values     |
| --------------- | --------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------- |
| `user_name`     | `string`  | The desired username for the new account.                                                                                                                                                                                                                                   |                     |
| `user_auth`     | `string`  | The user's email or phone number.                                                                                                                                                                                                                                           |                     |
| `auth_jwt`      | `string`  | An authentication JWT, if using a third-party authentication provider like Apple or Google.                                                                                                                                                                                 |                     |
| `auth_jwt_type` | `string`  | The type of authentication JWT provided.                                                                                                                                                                                                                                    | `apple`, `google`   |
| `password`      | `string`  | The desired password for the new account. This is required if `auth_jwt` is not provided.                                                                                                                                                                                   |                     |
| `network_name`  | `string`  | The desired name for the new network. It must satisfy: <ul><li>at least 3 characters different from an existing network name</li><li>at least 8 characters</li><li>domain name compatible (see [RFC 5890](https://datatracker.ietf.org/doc/html/rfc5890))</li><li>not contain the dash (-) character (BringYour subdomains are reverse-flattened to a single level using dashes)</li></ul> |                     |
| `terms`         | `boolean` | User's consent to accept the terms of service. Must be `true`.                                                                                                                                                                                                              | `true`              |

### Response

#### `200 Success`

**Status:** `200 OK`

**Content-Type:** `application/json`

A successful response indicates that the network has been created and may include details for further verification or network information.

```json
{
  "network": {
    "by_jwt": "text",
    "network_id": "text",
    "network_name": "text"
  },
  "verification_required": {
    "user_auth": "text"
  },
  "error": {
    "message": "text"
  }
}
```

| Field                       | Type     | Description                                                                           |
| --------------------------- | -------- | ------------------------------------------------------------------------------------- |
| `network`                   | `object` | Details about the newly created network.                                              |
| `network.by_jwt`            | `string` | A JWT for network authentication.                                                     |
| `network.network_id`        | `string` | The unique identifier for the new network.                                            |
| `network.network_name`      | `string` | The name of the new network.                                                          |
| `verification_required`     | `object` | Indicates if an additional verification step is needed.                               |
| `verification_required.user_auth` | `string` | The user authentication detail for which verification is required.                    |
| `error`                     | `object` | An optional error object, typically for informational messages.                       |
| `error.message`             | `string` | A descriptive message if an error or informational note is present.                   |
